```http

  "startLatitude": 31.2310,
  "startLongitude": 121.4745,
  "startAddress": "世纪公园南门",

  "plannedEndTime": "2026-04-21T17:00:00",

  "hasGuideDogThisRun": true
}
```

> `plannedStartTime` 和 `plannedEndTime` 必须是未来时间。
> `expectedDurationMinutes`：10-300 分钟。
> `hasGuideDogThisRun`：null 表示使用盲人档案默认值。

#### Step 9：查询订单状态（等待 1-2 秒）

```http
GET http://127.0.0.1:8081/api/orders/{{order_id}}
Authorization: Bearer {{blind_token}}
```

预期：`status: "PENDING_MATCH"`

> **说明**：串行派单中，订单始终处于 `PENDING_MATCH` 状态，直到有志愿者接受。`OrderDetailResponse` 不暴露 `dispatchCurrentVolunteerId`（内部 Redis 状态）。

#### Step 10：验证志愿者 2 未收到通知

通过 WebSocket 客户端连接志愿者 2（或查看服务日志），确认志愿者 2 没有收到此订单的派单通知。

> WebSocket 连接方式：`ws://127.0.0.1:8081/ws/volunteer?token={{volunteer_2_token}}`
>
> 只有当前被派到的志愿者会收到 `NEW_ORDER` 推送。

#### Step 11：志愿者 1 接单

```http
POST http://127.0.0.1:8081/api/orders/{{order_id}}/respond
Authorization: Bearer {{volunteer_1_token}}
Content-Type: application/json

{ "action": "ACCEPT" }
```

预期：`{ "success": true, "orderId": ... }`，订单状态变为 `PENDING_ACCEPT`。

> **旧接口兼容**：`POST /{id}/accept` 和 `POST /{id}/reject` 仍然可用，但推荐使用 `/respond`。

## 附：常见问题排查

| 现象 | 最可能的原因 | 检查位置 |
|------|-------------|---------|
| 订单提交后无志愿者被派到 | 候选队列为空（硬性过滤掉所有志愿者） | `ScoringService.scoreCandidates()` 的距离/导盲犬/注册状态过滤 |
| 志愿者拒绝后未转给下一人 | 定时任务未运行，或 Redis key 未正确过期 | `DispatchScheduler.processDispatchQueue()`、Redis TTL |
| 两人同时接单都成功 | 乐观锁未生效 | `RunOrder` 的 `@Version` 字段、`OrderService.acceptOrderWithRetry()` |
| 导盲犬订单派给了不接受导盲犬的志愿者 | 硬性过滤逻辑未执行 | `ScoringService` 中 `acceptsGuideDog` 过滤条件 |
| 超时后未自动转派 | 定时任务间隔配置问题 | `@Value("${app.dispatch.scheduler-poll-interval-ms}")` |
| 下单失败 "请先设置紧急联系人" | 盲人未添加紧急联系人 | `EmergencyContactService.hasContacts()` |
| 志愿者接单失败 "请先完成志愿者注册流程" | 志愿者 `registrationStep` 非 `STEP_4_COMPLETED` | `VolunteerProfile.registrationStep` |
| macOS `curl localhost` 失败 | macOS DNS 解析问题 | 使用 `127.0.0.1` 代替 `localhost` |

## 附：评分算法参考

ScoringService 5 维评分（权重之和 = 100%）：

| 维度 | 权重 | 说明 |
|------|------|------|
| 距离 | 40% | 越近分越高，超距离阈值 = 0 |
| 时间重叠 | 25% | 志愿者可用时间与订单时间的重叠率 |
| 评分 | 20% | 历史 avgRating（1.0-5.0），无评分默认 0.7 |
| 接单率 | 10% | acceptanceRate（0.0-1.0），无记录默认 0.7 |
| 配速匹配 | 5% | 盲人配速偏好与志愿者配速范围的匹配度 |

**硬性过滤条件**（任一不满足直接排除）：
1. 志愿者在线（Redis 有位置记录）
2. 距离在当前轮次搜索半径内
3. `registrationStep == STEP_4_COMPLETED`
4. 导盲犬兼容性（`hasGuideDogThisRun=true` 时要求 `acceptsGuideDog=true`）
5. 无可用时间时跳过时间过滤（不排除）
